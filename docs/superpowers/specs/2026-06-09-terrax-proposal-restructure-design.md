## 1. 背景与目标

`proposal.md` 内容扎实但"散"，根因是把"成文大纲(§4)"与"实验方案(§5)"两套体系平行堆叠，导致同一概念（指标、基线、数据）被定义两遍且口径不一。`Thoughts.docx` 则是一段已被导师打磨成型的漏斗式引言，逻辑脉络清晰。

**目标**：以 docx 的漏斗脉络为骨架，将 proposal 的方法与实验素材重新归位、去重，产出一份结构清晰、单一权威来源、话语统一的研究方案。

## 3. 写作约定（贯穿全文，硬性）

> 来源：用户与导师的复盘。用户写作/推导有"跳步直奔结论"的习惯，导师两次批其问题"太大/散"。docx 引言之所以好，靠的是**漏斗式收束 + 排比展开**。

1. **漏斗不可省**：凡引出问题处，按"宽→窄"显式递进（空间分析 → 大模型驱动的空间分析 → 空间分析智能体 → 提出问题），不得直接抛结论。
2. **排比即展开**：把抽象结论拆成读者可逐一点头的并列项（如 docx 的"对象遗漏、关系误解、约束缺失、选错工具、顺序错、依赖断裂、幻觉调用"）。docx 已有排比处沿用，新写处主动构造。
3. **补全推导桥**：显式写出 because→therefore 的过渡句与连接逻辑，宁可多写连接句也不跳步。
4. **话语统一**：术语以 docx 体系为准，全文一致。

## 4. 章节树（方案 A，含素材来源标注）

**一、研究背景与问题** ←《采用 docx 漏斗，替换 proposal 原绪论》
- 1.1 背景：空间分析是 GIS 核心；传统流程门槛高、固定工作流失效〔docx 段1〕
- 1.2 机遇：LLM 多智能体降低门槛（GIS Copilot、GeoAgent）〔docx 段2〕
- 1.3 问题①：端到端生成之弊（七项排比）〔docx 段3〕
- 1.4 问题②：空间规划的特殊性——多因素约束下 NL→空间操作语义对齐；Reviewer 后验修补无法回溯规划层；规划输出是面向人类文档而非面向智能体数据结构〔docx 段4〕
- 1.5 研究问题凝练 + 方法总思路：分层规划"NL任务→空间语义约束→工作流图→工具流"〔docx 段5〕
- 1.6 研究意义〔docx 段6〕

**二、研究目标与研究问题** ←〔proposal §一、§二〕
- 2.1 总目标
- 2.2 研究问题：
  - **RQ1（知识）**：面向任务规划的知识表示框架，统一存储工具功能特征、参数、环境依赖与兼容性约束。〔≈原 RQ1〕
  - **RQ2（规划·核心）**：多因素约束下 NL→空间语义约束→DAG 的两步结构化建模，输出含数据流依赖、顺序约束、可验证的执行计划。〔原 RQ2 ＋ docx 语义层〕
  - **RQ3（验证·反转）**：基于结构化表示的执行错误可定位与定向修正——利用语义约束+DAG 的可映射性，将执行失败精准回溯到规划层缺陷并修正。〔原 RQ3 重写〕
- 2.3 论点与假设（保留 proposal 三条，措辞统一）

**三、国内外研究现状** ←〔proposal §四.2 扩写 ＋ 8 篇文献〕
- 3.1 GeoAI：深度学习 × GIScience
- 3.2 LLM4GIS：大模型对 GIScience 的影响
- 3.3 Autonomous GIS：架构演进（原型→线性→单体→多智能体）、工具演进、规划层演进（一步到位→转嫁修正→中间流程分离但存在性/职责未定论）
- 3.4 规划层与错误处理评述（引出本文）；8 篇文献定位见本 spec 第 6 节；**点明 GeoAgent(Lin 2026) 标杆/问题源/对比基线三重身份**

**四、研究内容与技术路线** ←〔proposal §四.3 全进；L2 插入语义约束层〕
- 4.1 总体设计：L1–L4 架构表 ↔ RQ
- 4.2 **L1 规划知识层**（RQ1）：任务模式库 ＋ 工具能力描述（输入类型→操作→输出类型签名，平台无关映射）；存储 Neo4j / JSON
- 4.3 **L2 任务规划层**（RQ2·核心）= 两步法
  - 4.3.1 第一步 NL→空间语义约束表示（一等公民）：schema 含 分析目标 / 空间对象 / 空间关系 / 正约束 / 负约束 / 属性条件 / 输出要求 / 隐含语义操作化
  - 4.3.2 第二步 语义约束→DAG：节点（task_id/description/tool/input_schema/output_schema）＋数据流边＋全局约束；两阶段约束生成（粗→细）＋硬约束（无环、上游供给）
  - 4.3.3 DAG 验证器（networkx）：无环 / 类型匹配 / 约束满足
- 4.4 **L3 执行代理层**（RQ3 前段）：执行跟踪与错误捕获（异常类型 / 任务状态 / 关键数据状态）
- 4.5 **L4 规划迭代优化层**（RQ3·反转核心）：Plan Auditor 三步（诊断/定位/修正），强调"语义约束+DAG 使回溯成为可能"；修正知识积累（few-shot）

**五、实验设计与验证** ←〔proposal §四.x ＋ §5 合并去重，单一来源〕
- 5.1 数据条件〔合并 §四.1 与 §5.1.1〕

- 5.3 测试任务集：T×R 矩阵，90 任务
- 5.4 评估指标〔**合并 §四.3 的 5 个 与 §5.2.1 的 14 个 → 一套**：过程指标 / 结果指标 / 全局指标〕
- 5.5 对比基线与消融〔**合并 §四.4 与 §5.2.2 → 一份**：原版 / GeoAgent / GPT-4o 直生 / 本方法〕
- 5.6 实验日志格式〔proposal §五.3〕
- 5.7 预期结果〔proposal §四.5〕；强调反转后"结构化回溯可恢复 60–80% 的 CRS/顺序/类型类失败"

**六、创新点与预期成果** ←〔proposal §三 重整为 5 点〕
1. 面向任务规划的 DAG 显式依赖建模（解决输出不可计算、依赖丢失）
2. **NL→空间语义约束的结构化表示（新）**——多因素约束下的语义对齐

5. 真实地理数据环境（PostGIS＋GPKG＋轨迹）验证
